# RetroAggregator: CLAUDE.md

Сводная инструкция по разработке, сборке и архитектурным правилам для проекта Retrospective Aggregator.

## 🛠️ Сборка и запуск (Frontend)

Все команды выполняются в директории `frontend/`.

- **Версия Node**: v24.x (управление через NVM: `nvm use 24`)
- **Запуск dev-сервера**: `npm run dev` (запускается на `http://localhost:8090/retrospective/`)
- **Запуск туннеля**: `npm run tunnel` (проксирует порт `8090` через SSH на удаленный сервер)
- **Сборка проекта**: `npm run build` (выполняет сборку TypeScript и бандл через Vite)
- **Проверка типов**: `npx tsc -b`

## 🎨 Стайлгайд и оформление кода

### 1. Стилизация (CSS)
- Используем **Vanilla CSS** и CSS Custom Properties (CSS-переменные в `index.css`).
- Вся дизайн-система завязана на переменные с поддержкой двух тем (темная по умолчанию, светлая активируется через атрибут `html[data-theme="light"]`).
- Для карточек используем полупрозрачные glassmorphism-панели (`.glass`, `.glass-elevated`). 
- Избегайте хардкода цветов в CSS-файлах компонентов — всегда ссылайтесь на переменные из `index.css`.

### 2. Тултипы
- Для отображения тултипов используется HTML-атрибут `data-tooltip="..."`.
- Тултипы в шапках и верхних меню обязательно снабжаются CSS-классом `tooltip-bottom` (чтобы они всплывали снизу элемента и не улетали за границы экрана).

### 3. TypeScript / React
- Соблюдайте строгость типов. При сборке неиспользуемые импорты вызывают ошибку компиляции (настроено в `tsconfig.json`).
- Импортируйте типы явно через `import type`.
- Глобальные состояния (тема, данные текущего пользователя) хранятся в контекстах React (`ThemeContext`, `AuthContext`) в `App.tsx`.

### 4. Git
- Вся конфигурация игнорирования файлов централизована в корневом `.gitignore`.
- Локальные `.gitignore` внутри поддиректорий коммитить запрещено.
